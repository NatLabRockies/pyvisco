v2.1.3 (2026-06-29)
===================

Infrastructure release. Adds an automated rebuild of the Voila web app
on every published GitHub release. No library or API changes.

Added
-----

- ``.github/workflows/rebuild-hf-space.yml``: GitHub Actions workflow
  that triggers a factory rebuild of the
  `pyvisco-voila <https://huggingface.co/spaces/mspringer-nlr/pyvisco-voila>`_
  Hugging Face Space whenever a new release is published on GitHub.
  The Space clones ``NatLabRockies/pyvisco`` at Docker build time, so a
  factory rebuild is what propagates a new pyvisco release to the live
  web app and keeps the deployed version in lockstep with the latest
  tag. ``workflow_dispatch`` is exposed for occasional manual hotfixes
  between releases. The workflow authenticates to the Hugging Face Hub
  via the ``HF_TOKEN`` repository secret (a Hugging Face user access
  token with **write** scope on the Space) and calls
  ``huggingface_hub.HfApi.restart_space(factory_reboot=True)``.
